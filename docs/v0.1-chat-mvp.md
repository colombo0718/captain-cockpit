## 一、為什麼做（Why）

### 痛點

```
v0.0.2 的 Resume in Terminal 流程：
  點卡 → 開 terminal → claude --resume <sid> → 在 CLI 裡打字
   問題：
   1. CLI 介面醜、訊息流不易讀（無 markdown 渲染、tool call 雜訊多）
   2. 多 agent 並行時、每個都一個 terminal、來回切痛苦
   3. cockpit 看不到 agent 在做什麼（idle / thinking / waiting）
   4. 無法跨卡片視覺通知（agent X 等你回答時、卡片不會閃）
```

colombo 原話：「**claude code 在 terminal 介面的使用體驗真的沒有很好。如果不是 vs code 有出可以在漂亮介面用你的外掛、我是真的會錯過你**」

→ Cockpit 的價值不只是「組織樹檢視」、是**讓 LL 治理鏈在漂亮介面裡跑起來**。

### 戰略對齊

- 對應 memory [`human-ai-system-triangle`](../../matrix-manager/../.claude/projects/c--Users-USER-leaflune/memory/project_human_ai_system_triangle.md)：**人類層 = GUI / NL、AI 層 = agent、系統層 = CLI / process**——cockpit 是人類層的具體化
- 對應 memory [`register-solves-meeting-room-gap`](...)：員工不只活在文件、也活在 session、cockpit 讓 colombo 跟員工的對話從 terminal 移到 GUI

## 二、做到什麼程度算完成（Scope / Definition of Done）

### v0.1.0 MVP（單聊版）

**必須有**：

- [ ] 詳情面板新增「💬 Open Chat」按鈕（Resume in Terminal 保留為 fallback）
- [ ] 點 Open Chat → 詳情面板下半部變成聊天區（訊息流 + 輸入框）
- [ ] Extension 後端 spawn `claude --print --output-format stream-json --resume <sid>` 長駐 process
- [ ] stream-json events 逐行解析、推進 webview
- [ ] 訊息渲染：
  - [ ] user 訊息（藍底、靠右）
  - [ ] assistant text（純背景、靠左、markdown 渲染）
  - [ ] tool_use（摺疊卡片：「📖 Read PROJECT.md」、可展開看 input）
  - [ ] tool_result（摺疊輸出、預設只顯示前 5 行 + 展開鈕）
- [ ] 輸入框（textarea、Enter 送出、Shift+Enter 換行）
- [ ] Streaming typewriter（agent 邊吐字邊顯示、不等整段完成）
- [ ] 同 agent 不重複 spawn process（已開過的 chat 直接顯示既有 process 的訊息）
- [ ] 關 chat 面板 → kill child process（避免殭屍）

**先不做**（v0.1.1+）：

- 多 agent 同時 chat（grid / tabs）
- 卡片狀態指示燈
- 卡片閃爍提醒
- Permission 彈窗（走 fallback：先在 settings.json 預先白名單）
- 對話歷史顯示（先只看當下這次 chat、想看舊紀錄按 fallback Resume in Terminal）
- 訊息搜尋
- 訊息匯出 / 複製單則

## 三、技術架構

### 資料流

```
[Webview UI]
  │ (postMessage: send-text)
  │
  ▼
[Extension host]
  │ chat session manager
  │   ├── 為每個 agent 維護一個 child process
  │   └── 路由 stdin / stdout 跟 webview
  │
  ▼ stdin: 用戶輸入文字
  ▼ stdout: stream-json events（逐行）
  │
  ▼
[child process: claude --print --output-format stream-json --resume <sid>]
  │
  ▼ HTTPS
  │
  ▼
[Anthropic API、Max 訂閱實付 $0]
  │
  ▼ 同步寫
[~/.claude/projects/<encoded-cwd>/<sid>.jsonl]
```

### 核心 class 設計

```typescript
// src/chat/chat-session.ts
class ChatSession {
  agentId: string;          // <repo>/<sid8>
  sessionId: string;        // full UUID
  cwd: string;              // <repo> 完整路徑
  process: ChildProcess | null;
  messages: Message[];      // 當前 session 已收到的訊息
  status: 'idle' | 'thinking' | 'waiting-input' | 'error';
  
  start(): Promise<void>;   // spawn process
  send(text: string): void; // write to stdin
  stop(): void;             // kill process
  
  on('message', (msg: Message) => void);
  on('status-change', (status: string) => void);
}

// src/chat/chat-manager.ts
class ChatManager {
  sessions: Map<string, ChatSession>;
  getOrCreate(agent: Agent): ChatSession;
  closeAll(): void;
}
```

### Webview ↔ Extension 訊息協定

| Webview → Extension | Extension → Webview |
|--------------------|--------------------|
| `{ command: 'open-chat', agentId }` | `{ command: 'chat-opened', agentId }` |
| `{ command: 'send-text', agentId, text }` | `{ command: 'message', agentId, message }` |
| `{ command: 'close-chat', agentId }` | `{ command: 'status', agentId, status }` |

## 四、Stream-json Event 處理對照表

從觀察 `claude --print --output-format stream-json` 已知的事件類型（v0.1.0 處理範圍）：

| event type | 行為 |
|-----------|------|
| `system.init` | 標記 chat 已初始化、UI 切到 ready 狀態 |
| `assistant.message.content[*].text` | 渲染為 assistant 訊息（streaming append） |
| `assistant.message.content[*].tool_use` | 渲染為摺疊工具卡 |
| `user.message.content[*].tool_result` | 接到上一個 tool_use 卡裡顯示結果 |
| `result.success` | 標記 turn 結束、輸入框可再用 |
| `result.error` | 顯示錯誤訊息 + 重試按鈕 |
| 其他 | 忽略（v0.1.0 不處理） |

## 五、Permission 處理（v0.1.0 走 fallback）

v0.1.0 不在 webview 攔 permission、改用：

1. 在每個 agent 的 cwd 下、確保 `.claude/settings.json` 把常用 tool 預先白名單：
   ```json
   {
     "permissions": {
       "allow": [
         "Read(*)", "Glob(*)", "Grep(*)",
         "Edit(*)", "Write(*)",
         "Bash(git status:*)", "Bash(git diff:*)", "Bash(git log:*)"
       ]
     }
   }
   ```
2. 真的踩到不在白名單的 tool → child process 會 hang 等待回答 → 自動 timeout 30 秒、回 webview 顯示「需互動式批准、請改用 Resume in Terminal」

v0.1.3 才實作 webview 內 permission 彈窗。
